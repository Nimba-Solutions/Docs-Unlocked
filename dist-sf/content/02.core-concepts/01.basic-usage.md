# Basic Usage

Learn how to add and structure content in Docs Unlocked.

## Content Structure

Docs Unlocked uses a simple file-based content structure. Navigation is **automatically generated** at runtime from your directory structure:

### Numerical Prefixes for Ordering

Use numerical prefixes to control the order of sections and pages:

- **Sections**: `01.getting-started/`, `02.core-concepts/`, etc.
- **Pages**: `01.introduction.md`, `02.contributing.md`, etc.

The numbers are **not displayed** in the UI - they only control ordering. Files without prefixes will appear last.

## Adding New Pages

### 1. Create Markdown File

Create a new `.md` file in the appropriate directory with a numerical prefix:

```markdown
# My New Page

This is the content of my new page.

## Section Header

More content here...
```

**Example:**
- File: `public/content/03.api-reference/01.overview.md`
- Will appear as: "Api Reference" section ‚Üí "Overview" page

### 2. Navigation is Automatic!

That's it! The navigation is automatically generated when the app loads. No configuration files needed.

The app will:
- Extract the page title from the first H1 header (`# Page Title`)
- Group pages by directory (sections)
- Sort sections and pages by their numerical prefixes
- Follow internal links to discover more pages

### 3. Upload as StaticResource

When you build and deploy, all content files are automatically packaged into a ZIP StaticResource:

```bash
npm run build:sf
cci task run deploy --path force-app --org dev
```

The build process:
1. Scans `public/content/` directory
2. Packages all `.md` files into `docsContent.zip`
3. Deploys to Salesforce as a StaticResource

## Internal Links

Link to other pages using paths **without** the numerical prefixes:

```markdown
[Go to Configuration](/core-concepts/configuration)
[See Contributing Guide](/getting-started/contributing)
```

The app will automatically find the correct file (with or without prefixes) when following links.

### Images

Images are automatically loaded from your StaticResource. Use standard markdown syntax:

```markdown
![Alt text](/images/logo.png)
![Screenshot](/screenshots/app-overview.png "App Overview")
```

**Path Resolution:**
- Paths are relative to the `content/` directory in your StaticResource
- Example: `![Logo](/images/logo.png)` loads from `/resource/docsContent/content/images/logo.png`
- Absolute URLs (http/https) are used as-is

**Supported Formats:**
- PNG, JPG, JPEG, GIF, SVG, WebP

### Videos

Videos are automatically embedded when you use image syntax with a video file extension:

```markdown
![Demo Video](/videos/demo.mp4)
![Tutorial](/videos/tutorial.webm "How to use the app")
```

**Supported Formats:**
- MP4, WebM, OGG, MOV, AVI, WMV, FLV, MKV

**Video Features:**
- Automatic controls (play, pause, volume)
- Responsive sizing
- Accessible with aria-labels from alt text

## Best Practices

### File Naming

- Use lowercase with hyphens: `my-page.md`
- Use numerical prefixes for ordering: `01.my-page.md`
- Keep names descriptive but concise
- Prefixes are hidden in the UI

### Content Organization

- Keep pages focused on a single topic
- Use clear, descriptive headers
- Break up long content with sections
- Include code examples where helpful
- Use numerical prefixes consistently for predictable ordering

### Directory Structure
