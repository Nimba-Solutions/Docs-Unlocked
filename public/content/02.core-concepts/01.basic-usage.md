# Basic Usage

Learn how to add and structure content in Docs Unlocked.

## Content Organization

Docs Unlocked uses a simple file-based content structure. Your markdown files are organized in directories, and navigation is automatically generated from a manifest file.

### How Navigation Works

1. **Build Process** - When you run `npm run build:sf`, the build script:
   - Scans your `public/content/` directory
   - Extracts titles from markdown files (first H1 header)
   - Generates a `manifest.yaml` file with navigation structure
   - Packages everything into a ZIP StaticResource

2. **Runtime** - When the app loads:
   - Fetches `manifest.yaml` from the StaticResource
   - Builds navigation from the manifest
   - Displays pages based on the manifest structure

### Numerical Prefixes

Use numerical prefixes to control the order of sections and pages:

- **Sections**: `01.getting-started/`, `02.core-concepts/`, etc.
- **Pages**: `01.introduction.md`, `02.contributing.md`, etc.

**Important**: The numbers are **not displayed** in the UI - they only control ordering. Files without prefixes will appear last.

**Example:**
- File: `01.getting-started/01.introduction.md`
- Displays as: "Getting Started" section ‚Üí "Introduction" page
- Order: First section, first page

## Adding New Pages

### Step 1: Create Markdown File

Create a new `.md` file in the appropriate directory:

```markdown
# My New Page

This is the content of my new page.

## Section Header

More content here...
```

**Example:**
- Create: `public/content/03.api-reference/01.overview.md`
- Will appear as: "Api Reference" section ‚Üí "Overview" page

### Step 2: Rebuild and Deploy

After adding or modifying content:

```bash
npm run build:sf
cci task run deploy --path force-app --org dev
```

The build process will:
- Scan for new files
- Update the manifest
- Package everything into the ZIP StaticResource

### Step 3: Navigation Updates Automatically

That's it! The navigation is automatically updated based on the new manifest. No manual configuration needed.

## Internal Links

Link to other pages using paths **without** the numerical prefixes:

```markdown
[Go to Configuration](/core-concepts/configuration)
[See Contributing Guide](/getting-started/contributing)
```

The app will automatically resolve these paths using the manifest.

### Images

Images are automatically loaded from your StaticResource. Use standard markdown syntax:

```markdown
![Alt text](/media/logo.png)
![Screenshot](/media/app-overview.png "App Overview")
```

**Path Resolution:**
- Images in the `media/` folder: Use `/media/filename.jpg`
- Images in the `content/` folder: Use relative paths or `/images/filename.png`
- Example: `![Logo](/media/logo.png)` loads from `/resource/docsContent/media/logo.png`
- Absolute URLs (http/https) are used as-is

**Supported Formats:**
- PNG, JPG, JPEG, GIF, SVG, WebP

**Example:**

![Flan dessert](/media/flan-612x612.jpg "Delicious flan")

### Videos

Videos are automatically embedded when you use image syntax with a video file extension:

```markdown
![Demo Video](/media/demo.mp4)
![Tutorial](/media/tutorial.webm "How to use the app")
```

**Supported Formats:**
- MP4, WebM, OGG, MOV, AVI, WMV, FLV, MKV

**Video Features:**
- Automatic controls (play, pause, volume)
- Responsive sizing
- Accessible with aria-labels from alt text

### Navigation Cards

## Best Practices

### File Naming

- Use lowercase with hyphens: `my-page.md`
- Use numerical prefixes for ordering: `01.my-page.md`
- Keep names descriptive but concise
- Prefixes are hidden in the UI

### Content Organization

- Keep pages focused on a single topic
- Use clear, descriptive headers
- Break up long content with sections
- Include code examples where helpful
- Use numerical prefixes consistently for predictable ordering

### Directory Structure

## Search Functionality

Docs Unlocked includes powerful search capabilities:

- **Keyboard Shortcut**: Press `Ctrl+K` (or `Cmd+K` on Mac) to open search
- **Full-Content Search**: Searches through all page content, not just titles
- **Instant Results**: Results appear as you type
- **Highlighting**: Search terms are highlighted in results
- **Anchor Navigation**: Clicking a result navigates to the page and highlights the matching text
